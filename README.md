## what i'm working on

**[KohakuRAG_UI](https://github.com/matteso1/KohakuRAG_UI)** — Streamlit chat interface for a RAG pipeline built on the winning [WattBot 2025](https://www.kaggle.com/competitions/WattBot2025/overview) solution. Answers questions about AI's environmental impact using a curated corpus of energy/sustainability papers. I built the AWS Bedrock integration; my collaborator Blaise handled local deployment. This work is part of UW-Madison's Research Cyberinfrastructure (RCI) group under Chris Endemann.

> 🎤 **Talk**: [Deploying RAG in Bedrock vs. Local: WattBot 2025 Case Study](https://uw-madison-datascience.github.io/ML-X-Nexus/Applications/Videos/Forums/mlx_2026-02-17.html) — ML+X Forum, Feb 2026. Walked through cloud vs. local RAG tradeoffs with real performance/cost/latency comparisons.

**[madison-bus-eta](https://github.com/matteso1/madison-bus-eta)** — Real-time bus delay prediction for Madison Metro. XGBoost model with conformal prediction intervals so you get an honest uncertainty range, not just a point estimate.

**[ProjectGorgon](https://github.com/matteso1/ProjectGorgon)** — Speculative decoding engine for Llama-3-8B with custom CUDA kernels. Goal is >2x inference speedup using Medusa-style draft heads. This is where I taught myself GPU programming from scratch.
